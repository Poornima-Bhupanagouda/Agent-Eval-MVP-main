How Workflows Work

1. Create a Workflow
Click + New Workflow above. Give it a name, set your orchestrator's endpoint URL, and optionally add sub-agents.
2. Check Health
The platform pings each agent's health endpoint. Green = online, red = offline. Make sure your agents are running before evaluating.
3. Quick Test
Click Quick Test on a workflow card to send a single message and see the response instantly — no test cases needed.
4. Evaluate
Click Evaluate to run structured test cases. Paste JSON, upload a YAML file, or load a built-in sample. Each test shows a full per-metric breakdown with reasoning.
Test case format: [{"input": "What is the refund policy?", "expected": "30 days", "metrics": ["answer_relevancy", "faithfulness"]}]
